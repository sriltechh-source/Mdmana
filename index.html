🛒 Adding Store Products
📌
File: Part 3  |  Find: const PRODUCTS = [
Store products show "Today's Price" — price is asked via WhatsApp. Cart shows ₹0 for these.
1
Open Part 3, find the PRODUCTS array

Search for const PRODUCTS = [ — you'll see a list of objects, one per product.

2
Copy any existing product line and paste it at the end (before the ]; )

Copy one line exactly, paste it, then change each field:

// TEMPLATE — copy this, paste inside PRODUCTS array, change all values

{ id:601,          // ← Unique number. Use 600+ for new products. NEVER repeat!

  name:'Mango',       // ← Product name shown to customer

  img:'https://images.unsplash.com/photo-XXXXXXXX?w=400&h=400&fit=crop',

                  // ↑ Go to unsplash.com, search, right-click image → Copy image address

  category:'Fruits', // ← Must match a name in STORE_CATS array (see Section 3)

  badge:'Seasonal',  // ← Small label on image corner. Use '' for none

  badgeType:'orange',// ← 'green' | 'orange' | 'red'

  price:null,        // ← ALWAYS null for Store products (price via WhatsApp)

  priceNum:0,      // ← ALWAYS 0 for Store products

  desc:'Sweet Alphonso mangoes, freshly arrived from farm.',

  tags:['Fruits','Seasonal','Sweet'] // ← For search. Match category + keywords

},
3
Add the category to STORE_CATS if it's new

Find const STORE_CATS = [ in Part 3. Add your new category name inside the array: 

const STORE_CATS = ['All', 'Vegetables', 'Meat', 'Bakery', 'Groceries', 'Food', 'Fruits'];

//                                                                              ↑ Added
Field What it does Example
id Unique number. Never repeat. Use 600+ for new. 601
name Product name shown to customer 'Fresh Mango'
img Unsplash image URL (see tip below) 'https://images.unsplash.com/...'
category Must exactly match a name in STORE_CATS 'Fruits'
badge Small corner label. Use '' for no badge. 'Hot' / 'New' / 'Pure'
badgeType Badge colour 'green' / 'orange' / 'red'
price ALWAYS null for store products null
priceNum ALWAYS 0 for store products 0
desc Description shown in Quick View 'Fresh sweet mangoes'
tags Keywords for search. Include category. ['Fruits','Seasonal']
💡
Getting Unsplash images: Go to unsplash.com → Search your product → Click image → Right-click the big image → "Copy image address" → Paste it → Add ?w=400&h=400&fit=crop at the end if not already there.

🏪 Adding Market Products
📌
File: Part 3  |  Find: const MKT_PRODUCTS = [
Market products have REAL prices. Cart calculates the total and sends it in the WhatsApp order message.
1
Find MKT_PRODUCTS and add your product

There are two types — local (handmade/farm products) and second (2nd hand items). Add inside the correct section: 

// LOCAL PRODUCT TEMPLATE

{ id:611,               // ← Unique. Use 611, 612, 613... for local products

  name:'Fresh Cow Ghee',

  img:'https://images.unsplash.com/photo-XXXXXXXX?w=400&h=400&fit=crop',

  category:'Local',       // ← Keep as 'Local' for all local products

  type:'local',          // ← MUST be 'local' or 'second'

  badge:'Pure',

  badgeType:'green',

  price:'₹550/500g',    // ← Shown to customer exactly as written

  priceNum:550,         // ← Number only — used to calculate cart total

  desc:'Pure desi cow ghee made at home, traditional method.',

  tags:['Local','Ghee','Pure','Dairy']

},


// 2ND HAND PRODUCT TEMPLATE

{ id:711,               // ← Unique. Use 711, 712... for 2nd hand

  name:'Washing Machine',

  img:'https://images.unsplash.com/photo-XXXXXXXX?w=400&h=400&fit=crop',

  category:'Second',      // ← Keep as 'Second' for all 2nd hand

  type:'second',         // ← MUST be 'second'

  badge:'Working',

  badgeType:'green',

  price:'₹7,500',

  priceNum:7500,         // ← The number alone (no ₹ or comma)

  desc:'Samsung 6.5kg semi-automatic. Used 2 years. Good working condition.',

  tags:['Second','Appliances','Washing']

},
2
Add location and category mapping for the new product

Find const PRODUCT_LOCATIONS = { and const PRODUCT_CAT_MAP = { in Part 3. Add entries for your new product id: 

// In PRODUCT_LOCATIONS — which area is the seller from?

const PRODUCT_LOCATIONS = {

  // ... existing entries ...

  611:'Khanapur',    // ← your new local product

  711:'Nirmal',      // ← your new 2nd hand product

};


// In PRODUCT_CAT_MAP — what category does it fall under?

// For LOCAL: 'Grains & Rice','Oils & Fats','Spices','Sweets & Jaggery',

//            'Handmade','Honey & Natural','Vegetables','Handicraft','Cloth & Fabric'

// For 2ND HAND: 'Furniture','Electronics','Vehicles','Kitchen',

//               'Baby Items','Appliances','Books','Clothing','Sports'

const PRODUCT_CAT_MAP = {

  // ... existing entries ...

  611:'Oils & Fats',  // ← ghee = Oils & Fats category

  711:'Appliances',   // ← washing machine = Appliances

};
⚡
priceNum must be a plain number: If price is ₹1,200 write priceNum:1200 — no ₹, no commas, no spaces. This is used to calculate the cart total that goes into the WhatsApp message.

⚙️ Adding Services & Booking Forms
📌
File: Part 3  |  Find: BOOKING_CONFIGS and SERVICES_META
Every "Book Now" / "Apply Now" button opens a booking form. The form auto-captures GPS and sends all details to WhatsApp.
1
Add your service to BOOKING_CONFIGS (the actual form)

Find const BOOKING_CONFIGS = { in Part 3. Add a new entry before the closing }; 

// BOOKING FORM TEMPLATE — add inside BOOKING_CONFIGS

'car-wash': {        // ← your unique key (use in button onclick)

  title: 'Car Wash Booking',

  subtitle: 'Book a professional car wash',

  img: 'https://images.unsplash.com/photo-XXXXXXXX?w=600&h=300&fit=crop',

  fields: [

    { label:'Your Name',   id:'bName',    type:'text',     placeholder:'Full name' },

    { label:'Phone',        id:'bPhone',   type:'tel',      placeholder:'+91 XXXXXXXXXX' },

    { label:'Address',      id:'bAddress', type:'textarea', placeholder:'Pickup address' },

    { label:'Vehicle Type', id:'bVehicle', type:'select',

      options:['Bike','Car','SUV','Auto'] },       // ← dropdown menu

    { label:'Preferred Date',id:'bDate',   type:'date' },

    { label:'Special Notes',id:'bNotes',  type:'textarea', placeholder:'Any special request' },

  ]

},
2
Add the button that opens this booking form

In Part 2, anywhere you want the button:

<!-- Simple green button -->

<button class="btn-primary" onclick="openBooking('car-wash')">🚗 Book Car Wash</button>


<!-- WhatsApp green button -->

<button class="btn-primary btn-wa" onclick="openBooking('car-wash')">Book via WhatsApp</button>

The key 'car-wash' must exactly match what you used in BOOKING_CONFIGS.

3
Add it to SERVICES_META so it appears in search results

Find const SERVICES_META = [ in Part 3. Add your service:

{ label:'Car Wash Service',

  icon:'🚗',

  desc:'Professional car wash at your doorstep',

  bookingKey:'car-wash',   // ← same key as BOOKING_CONFIGS

  keywords:['car','wash','vehicle','clean','bike','auto']

},
Field type What it shows Use for
'text' Single line text input Name, vehicle number, item description
'tel' Phone number input (numeric keyboard) Phone numbers
'email' Email input Email addresses
'textarea' Multi-line text box Address, long descriptions, notes
'select' Dropdown menu Fixed choices (vehicle type, plan, etc.)
'date' Calendar date picker Booking date, delivery date
'time' Time picker Preferred time slot
'number' Number input Number of guests, quantity
💡
GPS is automatic! Every booking form already auto-captures the customer's GPS location and includes it in the WhatsApp message. You don't need to add anything for this.

🎨 Themes & Colours
📌
Part 1 (CSS variables) + Part 3 (THEMES array)
A
Change the default theme colour permanently

In Part 1, find :root { at the very top. Change the brand variables:

/* :root — these are the default colours shown on every fresh visit */

:root {

  --brand: #2D5016;          /* Main brand colour (buttons, links, chips) */

  --brand-dark: #7CB518;    /* Secondary brand (gradient end) */

  --brand-gradient: linear-gradient(135deg, #2D5016, #7CB518);

}
B
Add a completely new theme

Step 1: Add the CSS in Part 1 (after the existing theme blocks):

[data-theme="saffron"] {

  --brand: #FF6F00;

  --brand-dark: #FFB300;

  --brand-gradient: linear-gradient(135deg, #FF6F00, #FFB300);

}

Step 2: Add to THEMES array in Part 3:

{ name:'Saffron', key:'saffron', color:'#FF6F00' },

The theme picker will show it automatically.

🔍 Making Things Appear in Search
💡
Everything in these arrays is automatically searchable: Products (PRODUCTS + MKT_PRODUCTS), pages (PAGES_META), services (SERVICES_META), store categories (STORE_CATS).

Search appears automatically for:

  • New products — search uses name, category, tags[]
  • New pages — search uses label, desc, keywords[]
  • New services — search uses label, desc, keywords[]
  • New store categories — automatically included from STORE_CATS

To make search results more relevant, add more descriptive words to the tags or keywords arrays. Think about what words a customer might type — use those as tags. 

// Good tags — think like a customer:

tags:['Ghee','Cow ghee','Desi ghee','Pure ghee','Dairy','Fat','Butter']


// Good service keywords:

keywords:['car wash','vehicle cleaning','bike wash','clean car','auto wash']

ID numbering guide — never overlap these ranges:
Store products: 101–599  |  Local market: 600–699  |  2nd hand: 700–799
If you add more than 100 products in a range, just continue upwards (800+, 900+, etc.)
🚨
3 things that will break your site if wrong:
1. Duplicate id numbers in products — always use a new unique number
2. Category name mismatch — product.category must exactly match a name in STORE_CATS
3. Missing comma between objects in arrays — every } except the last one needs a , after it
✅ Before saving your changes — quick check:
  1. Every new product/item has a unique id not used anywhere else
  2. Every object in an array is followed by a comma (except the very last one)
  3. All text strings are wrapped in single quotes 'like this'
  4. Category names exactly match what's in the STORE_CATS / MKT_LOCAL_CATS / MKT_SECOND_CATS arrays
  5. Booking form key in button exactly matches the key in BOOKING_CONFIGS
  6. New page id is "page-yourkey" and PAGES_META key is "yourkey" (without "page-")
